feat: B2.1b Trezza Propiedades scraper

Local agency scraper following the same architecture as Zonaprop:
- lib/services/scrapers/trezza/urls.ts: partido slug to listing URL
- lib/services/scrapers/trezza/parser.ts: card parser + infinite scroll handling.
  Filters out Vendido/Alquilado/Reservado statuses and non-venta operations.
  Prices come from embedded JSON-LD RealEstateListing offers.
- lib/services/scrapers/trezza/index.ts: scrapeTrezza orchestrator
- scripts/scrape-trezza.ts: CLI

Schema:
- supabase/migrations/00003_add_trezza_source.sql: adds trezza to
  property_source enum

Persistence:
- Cached admin Supabase client (singleton) to avoid re-creating on every upsert

Verified end-to-end: 27 active Trezza venta listings persisted for Lomas
de Zamora. Combined with Zonaprop run, the partido now has 52 active
properties from 2 distinct sources — sets up B2.2 (cross-source dedup).

Known limit: ~5/27 properties miss the JSON-LD price because the script
tag is not always re-emitted after infinite scroll loads more cards.
Non-blocking, can be fixed by polling JSON-LD between scroll iterations.
